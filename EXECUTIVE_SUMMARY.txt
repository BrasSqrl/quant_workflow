Quant Studio
=============

Purpose
-------
Quant Studio is a model-development application for building, reviewing,
testing, and documenting quantitative models from tabular data. It is designed
to help teams move from raw development data to an auditable model package
without forcing the work into notebooks or ad hoc scripts.

Primary Users
-------------
Quant Studio is built primarily for:

- model developers who need a governed workflow for PD, LGD, CCAR, and CECL
  development
- model validation and risk teams who need transparent diagnostics,
  reproducible evidence, and structured review artifacts
- business stakeholders who need understandable summaries of what was built,
  how it was tested, and what was exported

Core Workflow
-------------
The application accepts a pandas dataframe, CSV file, or Excel file and guides
the run through the major stages of model development:

- ingestion and schema control
- target definition and feature setup
- imputation, governed transformations, and feature engineering
- advanced imputation, governed transformations, interaction screening, and
  feature-construction workbench review
- train, validation, and test splitting
- model-suitability and assumption checks
- variable selection and manual review
- model training, existing-model scoring, or feature-subset comparison
- calibration, evaluation, backtesting, and scenario testing
- expanded statistical-review frameworks for distribution shifts, residual
  bias, outliers, dependency clustering, econometric checks, and structural
  breaks
- feature-effect explainability outputs including partial dependence, ICE,
  centered ICE, accumulated local effects, two-way effect heatmaps, confidence
  bands, monotonicity diagnostics, segmented effects, effect stability,
  marginal effects, interaction strength, and feature-bucket calibration
- robustness and stability testing on repeated train resamples
- preset-aware workflow guardrails and readiness checks
- credit-risk-specific development diagnostics such as vintage, migration,
  recovery, and macro-sensitivity views
- visualization, documentation, and artifact export
- performance-aware export profiles, batched explainability scoring, and
  run-level debug traces for large-run review

Supported Development Scope
---------------------------
Quant Studio is intentionally narrow in scope. It focuses on model
development, challenger review, scenario analysis, validation support, and
documentation. It is not positioned as an ongoing production monitoring or
MLOps platform.

The current model-development scope supports probability of default as the
primary use case while also covering adjacent workflows commonly needed for
LGD, CCAR, and CECL development.

Model Families
--------------
Quant Studio currently supports a range of model families used in credit and
forecast-development workflows, including:

- logistic regression
- discrete-time hazard models
- elastic-net logistic regression
- scorecard logistic regression
- probit regression
- linear regression
- beta regression
- two-stage LGD models
- panel regression
- quantile regression
- Tobit regression
- XGBoost

The platform can also load a previously exported `quant_model.joblib` artifact
to score new data and rerun documentation, diagnostics, and validation outputs
without rebuilding the model.

It also includes a separate feature-subset-search mode that compares candidate
feature sets for a selected model family before a final development run is
chosen.

Key Outputs
-----------
Each run can produce a structured output package that includes:

- fitted model artifacts and scored outputs
- comparison-only subset-search bundles focused on candidate ranking, ROC/AUC,
  KS, feature frequency, frontier charts, and paired significance tests
- metrics, statistical tests, backtests, and validation tables
- interactive charts and exported HTML reporting
- a guided-versus-advanced GUI workflow so standard runs stay compact while
  specialist controls remain available when needed
- a thin Streamlit entrypoint with shared UI modules for state, workspace,
  results, caching, and config assembly
- challenger-comparison results
- robustness and stability summaries
- advanced imputation summaries and sensitivity comparisons
- multiple-imputation pooling summaries and Little's MCAR evidence
- scorecard binning workbench outputs
- feature-construction workbench outputs
- preset-aligned recommendation tables for imputation, transformations, and
  statistical tests
- scenario-testing outputs
- documentation packs and validation packs
- committee-ready and validation-ready DOCX/PDF reports
- normalized numerical warning capture and estimation-health diagnostics
- polished regulator-ready reports with cover pages, section maps, section
  summaries, split appendices, and explicit numerical-stability sections
- structured reference example packs for PD, LGD, CECL/lifetime PD, and CCAR
  forecasting workflows
- reproducibility manifests and rerun-ready code bundles
- run debug traces showing per-step timing, status, and shape changes
- fast, standard, and audit export profile support
- versioned monitoring handoff bundles for the separate monitoring application
- machine-readable artifact manifests that index the full output package
- Excel templates and governance workbooks for offline review

Transparency And Auditability
-----------------------------
Quant Studio is designed to make modeling decisions visible rather than hiding
them behind GUI actions or opaque scripts. Current transparency and auditability
features include:

- feature dictionaries with business definitions, lineage, and rationale
- governed transformations that are explicitly configured and exported
- advanced imputation rules that are explicitly configured, fitted on the
  training split, and exported
- lightweight performance safeguards that keep large-run previews and exported
  HTML reports usable
- batched explainability scoring for PDP, ICE, ALE, marginal effects, and
  interaction diagnostics where practical
- run-level debug traces that make slow or failed pipeline steps easier to
  isolate during review
- manual feature-review decisions and scorecard bin overrides
- preset-aware workflow guardrails tied to the selected development use case
- repeated-resample stability testing for held-out metrics and feature effects
- deeper statistical-review frameworks for distributional, residual,
  dependency, paired model-comparison, stationarity, and structural-break
  analysis
- model-suitability and assumption checks before fitting
- reproducibility manifests with hashes, package versions, and environment
  details
- structured documentation and validation packs for reviewer handoff
- deterministic reference workflows for core use cases
- walkthrough-style example packs that explain how to read the exported
  artifact bundles for those reference workflows

Business Value
--------------
At an executive level, Quant Studio helps teams standardize model-development
practice, improve review readiness, and reduce the time required to assemble
evidence for internal governance, validation, and documentation.
