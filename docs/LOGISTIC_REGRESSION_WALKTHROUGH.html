Before You Start

This is the shortest reliable path to a working binary logistic model in Quant Studio.

Recommended first-pass settings
  • Keep Execution mode = fit_new_model.
  • Set Model type = logistic_regression.
  • Set Target mode = binary.
  • Use Cross-sectional.
  • Keep the default split of 60 / 20 / 20.
  • Leave diagnostics and exports on for the first run.
What you must change

The key schema edits are to mark default_status as target_source, mark as_of_date as date, and disable loan_id plus legacy_text_field.

Open the Column Designer and assign the correct roles.

This is the critical step. Quant Studio needs to know which column is the raw target source and which columns should stay out of the feature set.

Expected modeled features after preprocessing

annual_income, debt_to_income, utilization, delinquency_count, region, employment_type, and derived date parts from as_of_date.

Why this matters

If the target row is wrong, the run models the wrong outcome. If identifier fields stay enabled, the model can learn meaningless high-cardinality noise.

Safe first-pass choices

Run Quant Model Workflow

What happens next

Quant Studio validates the config, builds the train / validation / test split, fits the model, scores the splits, creates diagnostics, and writes the artifact folder.

What to expect

The app shows a spinner during execution. When the run finishes, the canvas switches into the organized results view and a timestamped run folder is written under artifacts.

8

Move from the sample walkthrough to your real data.

Once the bundled-sample flow works, repeat the same path with your own file. The control flow stays the same even though the schema and assumptions change.

What to change for a real dataset
  • Turn off Use bundled sample data.
  • Upload your CSV, Excel, or Parquet file, or select it from Data_Load/.
  • Rebuild the Column Designer roles around your actual target, date, and identifier columns.
  • Only change missing-value policy when you have a defined business rule.
  • Update the documentation fields so the export pack reflects the real model purpose.
Best practice

Keep the first successful real-data run as your baseline. It gives you a reproducible artifact folder, validation pack, and saved model object for future existing-model scoring.

One-page click checklist

  1. Launch Quant Studio with .\launch_gui.bat.
  2. Leave Use bundled sample data on.
  3. Set Execution mode = fit_new_model.
  4. Set Model type = logistic_regression.
  5. Set Target mode = binary.
  6. Set Data structure = cross_sectional.
  7. Open Column Designer.
  8. Mark default_status as target_source.
  9. Mark as_of_date as date.
  10. Disable loan_id and legacy_text_field.
  11. Leave missing policy on inherit_default.
  12. Click Run Quant Model Workflow.
  13. Review Overview, then Model Performance, then Calibration / Thresholds, then Governance.
  14. Keep quant_model.joblib, run_config.json, and interactive_report.html from the export folder.

This walkthrough is aligned to the current Quant Studio interface and to the bundled-sample logistic run used during documentation generation. Update it alongside the app if the GUI layout or control names change.
