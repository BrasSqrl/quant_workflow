Quant Studio SageMaker Setup
============================

Use this file when you want to run Quant Studio from a browser-based
SageMaker environment such as Code Editor or JupyterLab.

Recommended path
----------------

1. Clone the repo into the SageMaker space.
2. Open a terminal in the repo root.
3. Run:

   bash scripts/bootstrap_sagemaker.sh

4. Start the GUI:

   bash scripts/run_sagemaker_streamlit.sh

5. Open port 8501 through SageMaker's browser preview / forwarded-port feature.

JupyterLab proxy launch command
-------------------------------

If you are running from a SageMaker JupyterLab space and the normal proxy URL
returns 404 or a blank page, start the GUI with the JupyterLab absolute proxy
base path:

   BASE_URL_PATH=/jupyterlab/default/proxy/absolute/8501 bash scripts/run_sagemaker_streamlit.sh

Then open this URL pattern in the browser, using the same host and region from
your current JupyterLab tab:

   https://<your-space>.studio.<region>.sagemaker.aws/jupyterlab/default/proxy/absolute/8501/

Notebook instance proxy launch command
--------------------------------------

If you are running from a SageMaker Notebook Instance instead of Studio
JupyterLab, use this command:

   BASE_URL_PATH=/proxy/absolute/8501 bash scripts/run_sagemaker_streamlit.sh

Then open:

   https://<notebook-name>.notebook.<region>.sagemaker.aws/proxy/absolute/8501/

What the scripts do
-------------------

- scripts/bootstrap_sagemaker.sh
  - checks for Python 3.11+
  - creates a repo-local .sagemaker_venv virtual environment
  - installs the dependencies from requirements-sagemaker.txt into that venv
  - installs Quant Studio from the local source tree

- scripts/run_sagemaker_streamlit.sh
  - uses .sagemaker_venv automatically when it exists
  - runs Streamlit on 0.0.0.0:8501
  - accepts BASE_URL_PATH for SageMaker proxy paths
  - keeps the same upload-size settings used by the Windows launcher

Why the virtual environment matters
-----------------------------------

SageMaker JupyterLab and Code Editor images include their own Jupyter and
Notebook packages. Installing application dependencies directly into that base
environment can surface resolver warnings such as:

   notebook 7.4.4 requires jupyterlab<4.5,>=4.4.4, but you have jupyterlab 4.2.5

Quant Studio does not need to alter SageMaker's IDE packages. The bootstrap
script therefore installs the app into .sagemaker_venv so the Streamlit app
runtime is isolated from SageMaker's Jupyter runtime.

If direct pip install does not work
-----------------------------------

Your SageMaker domain may not have outbound access to PyPI, or it may require a
private package source. In that case, use one of these:

1. A SageMaker lifecycle configuration that clones the repo and runs the
   bootstrap script automatically.
2. A custom SageMaker image with the dependencies preinstalled.

Starter lifecycle script
------------------------

This repo includes:

   scripts/sagemaker_code_editor_lifecycle.sh

That script is a starter for SageMaker Code Editor lifecycle configuration
usage. It can clone or update the repo and then run the bootstrap script.

Files added for SageMaker support

- scripts/run_sagemaker_streamlit.sh

Recommended commands
--------------------

If the repo is already cloned:

   cd /home/sagemaker-user/quant_workflow
   bash scripts/bootstrap_sagemaker.sh
   BASE_URL_PATH=/jupyterlab/default/proxy/absolute/8501 bash scripts/run_sagemaker_streamlit.sh

If the dependencies are already built into the image:

   cd /home/sagemaker-user/quant_workflow
   BASE_URL_PATH=/jupyterlab/default/proxy/absolute/8501 bash scripts/run_sagemaker_streamlit.sh
